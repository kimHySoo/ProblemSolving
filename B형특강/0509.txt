정수 배열이 주어졌을 때, 계속 작업을 처리하는 것. 단발성으로 끝나는 것이 아니라 상황이 주어지면 그 상황에 계속 잡업을 하는것

오프라인 쿼리 -> 마지막에 한번에 답을 출력
이때, 모든 쿼리를 순서대로 처리할 필요없이 마지막에서 역순으로 처리가 가능하다.
언제 효율적? -> 간선을 끊는 대신 점점 연결하는 방식으로 진행

누적합 -> 업데이트 없는 쿼리에서만 사용 가능
일반 업데이트 O(1) get O(N)
누적합 업데이터 O(N) get O(1)

imos difference array -> lazy한 처리


기능별로 update, get 구분해보기 - init은 무조건 update
update -> 결과에 영향을 줌
get -> 결과에 영향을 주지 않음 
--> 이를 통해서 update get 최적화 방식 구분해 볼 수 있음

update가 느리다 -> 여기 저기 값을 바꿔줌 -> 바꿔야 할 데가 많다. -> 데이터를 지우는 시도(lazy한 처리) -> get이 느려짐
1. 진짜 필요해? -> 걍 안함
2. Lazy하게 처리 -> 지금 업데이트 안함

get -> 자기가 가져오기 편한데서 데이터 가져옴 -> 편하게 가져올 만한 정보가 없다. -> updater가 느려짐
이거 조율해 주는게 프로문제 특징-> 호출의 빈도를 보고 누가 짬맞을지 정함
