# Binary Lifting 오답노트

---

## 1. LCA 탐색 방향 — depth 맞출 때 올려야 할 노드 실수

### 틀린 코드
```java
if (dist[x] < dist[y]) {
    int tmp = x; x = y; y = tmp;
}
int diff = dist[x] - dist[y];
for (int j = 0; j < 17; j++)
    if (((diff >> j) & 1) == 1) y = parent[y][j]; // y를 올림 ❌
```

### 올바른 코드
```java
if (dist[x] < dist[y]) {
    int tmp = x; x = y; y = tmp;
}
int diff = dist[x] - dist[y];
for (int j = 0; j < 17; j++)
    if (((diff >> j) & 1) == 1) x = parent[x][j]; // x를 올림 ✅
```

### 이유
```
swap으로 x를 더 깊은 노드로 만들었으므로
올려야 할 노드는 x
y를 올리면 depth가 안 맞아짐
```

## 2. LCA 탐색 루프 — 변수 혼용 (i를 j로 써야 함)

### 틀린 코드
```java
for (int j = 16; j >= 0; j--) {
    if (parent[x][i] == parent[y][i]) break; // i는 바깥 루프 변수 ❌
    x = parent[x][j];
    y = parent[y][j];
}
```

### 올바른 코드
```java
for (int j = 16; j >= 0; j--)
    if (parent[x][j] != parent[y][j]) { // j로 통일 ✅
        x = parent[x][j];
        y = parent[y][j];
    }
```

### 이유
```
바깥 루프 변수 i는 1~n 범위
parent[][i]에서 i가 17 이상이면 배열 인덱스 초과
→ 런타임 에러
```

## 3. LCA 탐색 루프 — break 사용 실수

### 틀린 코드
```java
for (int j = 16; j >= 0; j--) {
    if (parent[x][j] == parent[y][j]) break; // 같으면 멈춤 ❌
    x = parent[x][j];
    y = parent[y][j];
}
```

### 올바른 코드
```java
for (int j = 16; j >= 0; j--)
    if (parent[x][j] != parent[y][j]) { // 다르면 올라감 ✅
        x = parent[x][j];
        y = parent[y][j];
    }
```

### 이유
```
        1  ← LCA
       / \
      2   3
     /
    4

lca(4, 3): depth 맞추면 x=2, y=3

break 방식:
j=16~1: parent[2][j]=1, parent[3][j]=1 → 같으니 break
→ 아무것도 안 올라감
→ x = parent[2][0] = 1 (우연히 맞는 경우도 있음)

더 복잡한 트리에서는 틀린 LCA 반환

올바른 방식:
"다르면 올라간다, 같으면 스킵"
큰 k부터 시도 → LCA를 넘지 않는 최대한 올라감
루프 끝나면 x, y는 LCA 바로 아래
→ parent[x][0] = LCA
```

## 4. LCA 탐색 루프 — 시작 인덱스 초과

### 틀린 코드
```java
for (int j = 17; j >= 0; j--) { // 17부터 시작 ❌
    if (parent[x][i] == parent[y][i]) break;
    ...
}
```

### 올바른 코드
```java
for (int j = 16; j >= 0; j--) { // 16부터 시작 ✅
    ...
}
```

### 이유
```
parent는 [n+1][17] 선언
인덱스 범위: 0 ~ 16
j=17로 시작하면 parent[][17] 접근 → 배열 인덱스 초과
→ 런타임 에러
```

## 5. 루트 노드 초기화 누락

### 틀린 코드
```java
for (int i = 2; i < n + 1; i++) {
    int x = Integer.parseInt(st.nextToken());
    node[x].child.add(node[i]);
    parent[i][0] = x;
}
// parent[1][0] 초기화 없음 ❌ → 기본값 0
```

### 올바른 코드
```java
parent[1][0] = 1; // 루트는 자기 자신 ✅
for (int i = 2; i < n + 1; i++) {
    int x = Integer.parseInt(st.nextToken());
    node[x].child.add(node[i]);
    parent[i][0] = x;
}
```

### 이유
```
전처리 시
parent[1][k] = parent[parent[1][k-1]][k-1]
             = parent[0][k-1]  // 0번 노드 접근
→ 런타임 에러

루트의 부모를 자기 자신(1)으로 설정해야
parent[0] 접근을 막을 수 있음
```

## 6. 재귀 DFS — 스택 오버플로

### 틀린 코드
```java
static void dfs(Node cur, int cnt) {
    dist[cur.num] = cnt;
    for (Node next : cur.child) dfs(next, cnt + 1); // 재귀 ❌
}
```

### 올바른 코드 (BFS로 교체)
```java
while (!q.isEmpty()) {
    Node cur = q.poll();
    turn[idx++] = cur.num;
    for (Node next : cur.child) {
        dist[next.num] = dist[cur.num] + 1; // ✅
        q.add(next);
    }
}
// dfs 메서드 삭제
```

### 이유
```
일자 트리일 때:
1 → 2 → 3 → ... → 100,000

dfs(1) → dfs(2) → dfs(3) → ... → dfs(100,000)
재귀 100,000번 중첩

Java 기본 스택 크기: 약 512KB ~ 1MB
재귀 한계: 약 5,000 ~ 10,000번
→ StackOverflowError

BFS는 재귀 없이 동일한 결과를 냄
```
