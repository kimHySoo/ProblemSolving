# 📌 DFS + DP 오답노트 (숫자 이동 문제)

## 🧠 문제 핵심

* NxN 격자
* 현재 위치에서 상하좌우 중 **값이 +1인 칸으로만 이동 가능**
* 가장 긴 이동 경로 길이와 시작 숫자 찾기

---

## ❌ 내가 했던 실수

### 1. ❌ 이동 조건에 dp 사용

```java
if(dp[nr][nc] != -1)
```

👉 문제점

* `dp`는 경로 길이 저장용
* 이동 가능 여부는 `map`으로 판단해야 함

👉 올바른 조건

```java
map[nr][nc] == map[r][c] + 1
```

---

### 2. ❌ 이미 계산된 칸을 막음

```java
dp[nr][nc] != -1 → 이동 불가
```

👉 문제점

* 메모이제이션에서는 이미 계산된 값은 재사용해야 함
* 이걸 막으면 경로가 끊김

👉 핵심 개념

> 방문 체크와 DP 캐싱은 완전히 다르다

---

### 3. ❌ += 사용 (경로 누적)

```java
dp[r][c] += dfs(nr, nc);
```

👉 문제점

* 이 문제는 갈 수 있는 방향이 최대 1개
* 누적이 아니라 단일 경로

👉 올바른 방식

```java
dp[r][c] = 1 + dfs(nr, nc);
```

---

### 4. ❌ idx 초기값 및 갱신 오류

```java
idx = 0;
idx = Math.min(idx, map[r][c]);
```

👉 문제점

* 항상 0 유지됨

👉 올바른 방식

```java
idx = Integer.MAX_VALUE;

if(len > max) {
    max = len;
    idx = map[r][c];
} else if(len == max) {
    idx = Math.min(idx, map[r][c]);
}
```

---

## ✅ 올바른 풀이 구조

### 1. DFS + 메모이제이션

```java
static int dfs(int r, int c) {
    if (dp[r][c] != -1) return dp[r][c];

    dp[r][c] = 1;

    for (int d = 0; d < 4; d++) {
        int nr = r + dr[d];
        int nc = c + dc[d];

        if (nr < 0 || nc < 0 || nr >= n || nc >= n) continue;

        if (map[nr][nc] == map[r][c] + 1) {
            dp[r][c] = 1 + dfs(nr, nc);
            break;
        }
    }

    return dp[r][c];
}
```

---

### 2. 전체 탐색

```java
for (int i = 0; i < n; i++) {
    for (int j = 0; j < n; j++) {
        int len = dfs(i, j);

        if (len > max) {
            max = len;
            idx = map[i][j];
        } else if (len == max) {
            idx = Math.min(idx, map[i][j]);
        }
    }
}
```

---

## 🔥 핵심 개념 정리

### 1. DP 의미 정의

```
dp[r][c] = (r,c)에서 시작했을 때 갈 수 있는 최대 이동 길이
```

---

### 2. DFS + DP 패턴

```
1. 이미 계산 → 바로 반환
2. 기본값 설정
3. 다음 상태 탐색
4. 결과 저장
```

---

### 3. 문제 구조

```
일반 DFS: 여러 갈래
이 문제: 단일 경로 (Linked List 구조)
```

---

### 4. += vs = 기준

| 상황    | 연산 |
| ----- | -- |
| 여러 경로 | += |
| 단일 경로 | =  |

👉 이 문제는 단일 경로 → '=' 사용

---

## 🧩 한 줄 핵심 정리

> 이 문제는 탐색 문제가 아니라 **연속된 숫자 수열 길이 구하기 문제**다

---

## 🚀 코테용 판단 기준

다음 조건 나오면 바로 떠올리기:

```
값 + 1
```

👉 특징

* 단일 경로
* DFS + DP
* '=' 사용

---

## ⭐ 추가 인사이트

* 방문 배열 없이도 해결 가능 (DP로 중복 제거)
* 그래프가 아니라 사실상 "연결된 수열" 문제
* 시간복잡도: O(N^2)

---

## 🏁 최종 정리

> 🔥 "DP는 이동 조건에 쓰는 게 아니라, 결과를 저장하는 데 쓰는 것이다"
