# 자료구조 시간복잡도 정리

## List (ArrayList)

| 연산 | 복잡도 | 설명 |
|---|---|---|
| 인덱스 접근 | O(1) | `list.get(3)` 배열 직접 접근 |
| 탐색 | O(N) | 처음부터 순서대로 확인 |
| 수정 | O(N) | 위치를 먼저 탐색해야 함 |
| 추가 (맨 뒤) | O(1) | 그냥 이어붙임 |
| 추가 (중간) | O(N) | 뒤 원소들 전부 한 칸씩 밀어야 함 |
| 삭제 (맨 뒤) | O(1) | 그냥 제거 |
| 삭제 (중간) | O(N) | 뒤 원소들 전부 한 칸씩 당겨야 함 |

### 수정이 O(1)인 경우

인덱스를 이미 알고 있을 때만 O(1)이다.

```java
list.set(3, "apple");       // 인덱스 직접 지정 → O(1)
// "banana"를 찾아서 수정  → 탐색 O(N) + 수정 O(1) = O(N)
```

## Set (HashSet)

### 내부 구조: 해시테이블

문자열을 해시함수에 넣어 배열 인덱스로 변환하고 그 위치에 바로 접근한다.

```
"apple" → hashCode() → 192837 → 192837 % 배열크기 → index 7
```

탐색 없이 한 번에 위치를 계산해서 바로 접근하므로 평균 O(1)이다.

### 충돌 해결: 체이닝

다른 값이 같은 해시를 가지면 같은 슬롯 안에서 연결해서 해결한다.

```
hashCode("apple")  = 7  ┐
hashCode("banana") = 7  ┘ 충돌

배열[7] → apple → banana → null
```

Java 8 이후 슬롯 내 원소 수에 따라 자동으로 자료구조를 전환한다.

```
충돌 8개 이하 → LinkedList → O(N)
충돌 8개 초과 → Red-Black Tree → O(log N)
```

### 시간복잡도

| 연산 | 복잡도 | 설명 |
|---|---|---|
| 탐색 | 평균 O(1) | 해시로 위치 바로 계산 |
| 추가 | 평균 O(1) | 해시로 위치 바로 계산 |
| 삭제 | 평균 O(1) | 해시로 위치 바로 계산 |
| 수정 | 없음 | 인덱스 개념이 없어 삭제 후 추가로 대체 |

평균 O(1)이지만 충돌이 많으면 최악 O(N)이 될 수 있다.
Java HashSet은 충돌을 잘 분산시켜 실질적으로 O(1)로 취급한다.

## List vs Set 비교

| 연산 | List | Set |
|---|---|---|
| 탐색 | O(N) | 평균 O(1) |
| 추가 | O(1) ~ O(N) | 평균 O(1) |
| 삭제 | O(1) ~ O(N) | 평균 O(1) |
| 수정 | O(N) | 없음 |
| 인덱스 접근 | O(1) | 없음 |
| 순서 보장 | O | X |
| 중복 허용 | O | X |

### 용도 구분

- 순서가 필요하거나 중복을 허용해야 하면 **List**
- 존재 여부 확인이 주목적이면 **Set** (contains가 O(N) → O(1))

```java
// N, M = 100000일 때
List.contains() // O(N × M) = 10^10 → 시간 초과
Set.contains()  // O(N + M) = 2×10^5 → 통과
```

## HashSet vs Trie 비교

| | HashSet | Trie |
|---|---|---|
| 구조 | 해시테이블 | 트리 |
| contains | 평균 O(1) | O(문자열 길이) |
| 용도 | 존재 여부 확인 | 접두사 탐색, 자동완성 |

문자열 존재 여부만 확인할 때는 HashSet이 더 빠르다.
접두사 탐색이나 자동완성이 필요할 때는 Trie를 사용한다.
