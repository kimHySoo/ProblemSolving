# KMP 알고리즘 (Knuth-Morris-Pratt)

## 개요

본문 문자열에서 패턴 문자열이 등장하는 위치를 효율적으로 찾는 알고리즘.
시간복잡도: **O(n + m)** (n = 본문 길이, m = 패턴 길이)

---

## 핵심 아이디어

일반 브루트포스는 불일치 시 본문 포인터 `i`를 뒤로 되감는다.
KMP는 `i`를 절대 뒤로 되감지 않고, 패턴 포인터 `j`만 PI 배열을 따라 점프한다.

---

## 전체 구조

```
1단계: 패턴으로 PI 배열 생성  →  getPi()
2단계: 본문에서 패턴 탐색      →  kmp()
```

PI 배열이 "지도" 역할을 하므로, 탐색 전에 반드시 먼저 만들어야 한다.

---

## 1단계 — PI 배열 (실패 함수)

### 정의

`PI[i]` = 패턴을 0~i까지 잘랐을 때, **접두사 = 접미사**인 최대 길이

```
패턴:  A B A B A C A
인덱스: 0 1 2 3 4 5 6
PI:    0 0 1 2 3 0 1
```

예) PI[4] = 3 → "ABABA"에서 앞 "ABA" = 뒤 "ABA"

### 동작 원리

`j`를 루프 **밖**에 선언해서 이전 상태를 유지하는 것이 핵심.

- `pattern[i] == pattern[j]` → `PI[i] = j + 1`, `j++`
- `pattern[i] != pattern[j]` && `j > 0` → `j = PI[j-1]` 로 점프, i 그대로
- `pattern[i] != pattern[j]` && `j = 0` → `PI[i] = 0`, i만 증가

### 코드

```java
public static int[] getPi(String p) {
    int[] pi = new int[p.length()];
    int j = 0;  // 루프 밖! 이전 상태 유지
    for (int i = 1; i < p.length(); i++) {
        while (j > 0 && p.charAt(i) != p.charAt(j))
            j = pi[j-1];
        if (p.charAt(i) == p.charAt(j))
            pi[i] = ++j;
    }
    return pi;
}
```

### 주의: j를 루프 안에서 초기화하면 안 된다

```java
// 틀린 예
for (int i = 1; i < p.length(); i++) {
    int j = i - 1;  // ← 매번 초기화하면 연속성이 끊김
    ...
}
```

i=2에서 j=1이 됐다면, i=3에서는 j=1을 이어받아야 한다.
초기화하면 그 정보가 날아간다.

---

## 2단계 — KMP 탐색

### 동작 원리

- `text[i] == pattern[j]` → `j++`
  - `j == m` (패턴 끝까지 일치) → 패턴 발견! `j = PI[j-1]`로 이동 후 계속
- `text[i] != pattern[j]` → `j = PI[j-1]`로 점프 (i는 그대로!)
  - `j == 0` 이면 더 물러날 곳 없음, i만 증가

불일치가 발생하면 일치할 때까지 또는 j=0이 될 때까지 while로 반복 점프한다.

### 코드

```java
public static int kmp(String t, String p) {
    int[] pi = getPi(p);
    int cnt = 0;
    int j = 0;
    for (int i = 0; i < t.length(); i++) {
        while (j > 0 && t.charAt(i) != p.charAt(j))
            j = pi[j-1];          // 불일치: j만 점프, i 그대로
        if (t.charAt(i) == p.charAt(j))
            j++;
        if (j == p.length()) {    // 패턴 발견
            cnt++;
            j = pi[j-1];          // 마지막 PI값으로 이동 후 계속
        }
    }
    return cnt;
}
```

---

## PI 배열이 점프 테이블인 이유

PI값은 "접두사=접미사 길이"인 동시에 "j를 이동할 위치"다.

```
j=5에서 불일치 발생
→ j = PI[4] = 3

PI[4]=3의 의미:
  "ABABA"에서 앞 3글자 = 뒤 3글자
  → 본문의 직전 3글자는 이미 패턴 앞 3글자와 일치함이 보장
  → j=3부터 이어서 비교하면 됨
```

---

## 시간복잡도가 O(n+m)인 이유 (분할 상환 분석)

`j`는 `++j`로만 증가하며 전체 루프에서 최대 n번 증가할 수 있다.
`while`문 안에서 j는 감소만 하는데, j는 0 밑으로 내려가지 않으므로
**감소 횟수는 증가 횟수(최대 n번)를 절대 넘을 수 없다.**

따라서 while이 특정 단계에서 많이 돌더라도, 전체 합산은 O(n).
getPi도 동일한 이유로 O(m). 합쳐서 **O(n + m)**.

---

## 다른 알고리즘과 비교

| 알고리즘 | 시간복잡도 (평균) | 시간복잡도 (최악) | 특징 |
|----------|-----------------|-----------------|------|
| 브루트포스 | O(n·m) | O(n·m) | i를 뒤로 되감음 |
| KMP | O(n+m) | O(n+m) | i 절대 불변, j만 점프 |
| Boyer-Moore | O(n/m) | O(n+m) | 오른쪽→왼쪽 비교, 실용적으로 가장 빠름 |
| Rabin-Karp | O(n+m) | O(n·m) | 해시 기반 |

두 문자열을 모두 최소 한 번은 읽어야 하므로 O(n+m)이 이론적 하한선.
KMP는 그 하한선에 정확히 닿아 있다.
