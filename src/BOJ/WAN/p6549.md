# 📊 히스토그램에서 가장 큰 직사각형 (스택 풀이)

## 🔥 문제 핵심
- 각 막대 높이가 주어짐
- 연속된 구간을 선택해서 만들 수 있는
- **최대 직사각형 넓이** 구하기

---

# ❌ 기존 풀이 (시간초과)

## 💡 방식
- 가능한 모든 높이에 대해
- 배열 전체를 순회하면서 넓이 계산

## 🚨 문제점
- 높이 종류 수 × 배열 길이 → **O(N^2)**
- 큰 입력에서 시간초과

---

# ✅ 정석 풀이 (스택, O(N))

## 💡 핵심 아이디어

> 각 막대가 **자기 높이로 만들 수 있는 최대 너비**를 구한다

---

# ⚙️ 동작 원리

- 스택에는 **인덱스**를 저장
- 스택은 항상 **높이가 증가하는 상태 유지**

---

## 📌 알고리즘 흐름

1. 현재 높이가 더 크면 → 스택에 push
2. 현재 높이가 더 작아지면 → pop 하면서 넓이 계산

---

## 📐 넓이 계산

```java
height = arr[top]
width = 현재 인덱스 - 이전 더 작은 높이 인덱스 - 1
area = height * width
```

---

# 🧠 코드

```java
import java.io.*;
import java.util.*;

public class Main {
    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

        while (true) {
            StringTokenizer st = new StringTokenizer(br.readLine());
            int n = Integer.parseInt(st.nextToken());

            if (n == 0) break;

            long[] arr = new long[n + 1];
            for (int i = 0; i < n; i++) {
                arr[i] = Long.parseLong(st.nextToken());
            }
            arr[n] = 0;

            Deque<Integer> stack = new ArrayDeque<>();
            long max = 0;

            for (int i = 0; i <= n; i++) {
                while (!stack.isEmpty() && arr[stack.peek()] > arr[i]) {
                    long height = arr[stack.pop()];

                    int left = stack.isEmpty() ? -1 : stack.peek();
                    long width = i - left - 1;

                    max = Math.max(max, height * width);
                }
                stack.push(i);
            }

            System.out.println(max);
        }
    }
}
```

# ❗ 중요 포인트

## 1. 각 막대는 한 번만 처리됨
- push 1번, pop 1번 → **O(N)**

---

## 2. 마지막 0 추가 이유
```java
arr[n] = 0;
```

👉 스택에 남은 막대를 강제로 처리하기 위해

---

## 3. long 사용 필수
- 넓이 = 높이 × 너비 → int overflow 가능

---

# 📊 예시

입력:
```
2 1 4 5 1 3 3
```

처리 흐름:
- 4,5 쌓임
- 1 등장 → 5,4 pop → 넓이 계산

> 스택으로 한 번에 계산하는 문제"

README 느낌으로 짧게 정리
병목 원인

높이 후보마다 전체 배열을 다시 탐색함

같은 막대를 여러 높이 기준으로 반복 확인함

시간복잡도 O(n^2)

해결 방향

높이별 탐색이 아니라 막대별 탐색으로 바꾼다

각 막대가 자기 높이로 확장 가능한 최대 구간을 구한다

더 낮은 막대를 만나는 순간 넓이를 계산한다

스택으로 각 막대를 1번 push, 1번 pop 하여 O(n)에 해결

필요한 생각

“모든 높이를 다 볼까?” 대신

“이 막대는 어디까지 유지될까?”를 묻기

중복 계산을 줄이는 방향으로 기준을 바꾸기
