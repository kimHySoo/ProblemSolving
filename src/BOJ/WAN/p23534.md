# 애너그램이 없는 최대 부분수열 개수

## 문제 설명

문자열 $s$의 부분수열들을 최대한 많이 뽑되, 뽑은 것들 중 **어떤 두 개도 애너그램이 되지 않아야** 합니다. 그 최대 개수를 구합니다.

- **애너그램**: 한 문자열을 문자 순서를 바꿔 다른 문자열로 만들 수 있을 때
- **부분수열**: 문자열에서 일부 문자를 순서를 유지하며 선택한 것

---

## 핵심 아이디어

> 두 부분수열이 애너그램 ⟺ **알파벳 구성(멀티셋)이 동일**

이 동치 관계를 이용해 문제를 변환합니다.

```
"애너그램이 없도록 부분수열을 최대한 뽑아라"
            ↓ 변환
"만들 수 있는 서로 다른 알파벳 멀티셋이 몇 개인가?"
            ↓ 변환
"각 알파벳 등장 횟수에 (횟수 + 1)을 모두 곱한 뒤 -1"
```

---

## 풀이 공식

문자열에서 알파벳 $c$가 $k_c$번 등장할 때, 부분수열에 해당 알파벳을 $0$~$k_c$번 포함시킬 수 있으므로 선택지는 $(k_c + 1)$가지입니다.

$$\text{답} = \prod_{c \in \text{알파벳}} (k_c + 1) - 1$$

빈 부분수열(모든 알파벳을 0번 선택)은 제외하므로 $-1$ 합니다.

### 예시

`s = "aab"` 라면:

| 알파벳 | 등장 횟수 | 선택지 |
|--------|-----------|--------|
| `a` | 2 | 3가지 (0개, 1개, 2개) |
| `b` | 1 | 2가지 (0개, 1개) |

$$3 \times 2 - 1 = \mathbf{5}$$

가능한 멀티셋: `{a}`, `{aa}`, `{b}`, `{ab}`, `{aab}` → **5개** ✓

---

## 구현 (Java)

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        String s = sc.next();

        int[] cnt = new int[26];
        for (char c : s.toCharArray()) {
            cnt[c - 'a']++;
        }

        long ans = 1;
        for (int i = 0; i < 26; i++) {
            if (cnt[i] > 0) {
                ans *= (cnt[i] + 1);
            }
        }

        System.out.println(ans - 1);
    }
}
```

| 항목 | 복잡도 |
|------|--------|
| 시간 | $O(n)$ |
| 공간 | $O(1)$ (알파벳 26개 고정) |

## 오답노트

### ❌ 흔한 실수

**실수 1. 완전탐색으로 모든 부분수열을 직접 나열**
- 길이 60짜리 문자열의 부분수열은 최대 $2^{60} \approx 10^{18}$개
- → **절대 불가능**, 수학적 변환이 필요

**실수 2. 애너그램 비교 로직을 직접 구현**
- 동치 관계를 이용해 문제를 멀티셋 개수 세기로 변환해야 함

**실수 3. 순서가 다른 부분수열을 다른 것으로 취급**
- `"ab"`와 `"ba"`는 애너그램 → **같은 그룹**으로 취급

**실수 4. `int`로 답을 저장**
- 최악의 경우 약 $3^{26} \approx 2.5 \times 10^{12}$ → `int` 범위 초과
- → 반드시 **`long`** 사용

**실수 5. 빈 부분수열을 포함**
- 곱셈 결과에서 반드시 **-1** 해야 함
