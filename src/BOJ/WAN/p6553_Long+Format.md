# 📌 Problem Solving 오답 노트

## 🔎 문제 요약
- 나라들이 원형으로 연결되어 있음 (사이클)
- 각 나라의 좌표가 주어짐
- 인접한 두 나라가 같은 거리로 이동하면서 최소 거리로 만날 수 있는 위치를 구해야 함

---

## 💡 핵심 아이디어
- 두 점에서 같은 거리 → 수직이등분선
- 그 중 최소 거리 → 두 점의 **중점 (midpoint)**

👉 따라서 모든 인접한 두 점의 중점을 구하면 됨

---

## ⚠️ 내가 틀린 이유

### 1. 입력 형식 착각
- 입력이 한 줄인지, 여러 줄인지 제대로 파악 안 함
- 실제로는 테스트 케이스마다 다양한 형식 가능

👉 해결:
```java
while ((line = br.readLine()) != null) {
    if (line.trim().isEmpty()) continue;
}
```

---

### 2. NumberFormatException 발생

#### 원인
- 좌표 범위: |x| < 10^12
- int 범위 초과 → 파싱 실패

#### 잘못된 코드
```java
int x = Integer.parseInt(...)
```

#### 해결
```java
long x = Long.parseLong(...)
```

---

### 3. 출력 형식 오류 (소수 6자리)

#### 해결 방법
```java
String.format("%.6f", value)
```

---

## 🧠 배운 점

### ✔️ 입력 먼저 의심해라
- 런타임 에러 → 입력 형식 문제일 확률 높음

### ✔️ 자료형 범위 항상 확인
- 문제 조건에 10^9 이상 나오면 무조건 long 의심

### ✔️ 출력 포맷 중요
- 실수 문제는 format 필수

---

## 🔥 최종 정리

```text
이 문제는 인접한 두 점의 "중점"을 구하는 문제
단, 좌표 범위 때문에 long 사용 필수
출력은 소수 6자리까지 format
```

---

## 🚀 한 줄 요약
👉 구현보다 입력 + 자료형 + 출력 형식이 핵심인 문제
