# 💎 보석 도둑 (그리디 + 우선순위큐)
- 그리디 문제 작성에서 에러
## 🔥 문제 핵심
- 가방마다 **최대 1개의 보석**만 담을 수 있음
- 각 가방에는 **무게 제한**이 있음
- 목표: **보석 가격 합 최대화**

---

# ❌ 내가 처음 접근한 방식 (틀린 이유)

## 💡 아이디어
- 보석을 **가격 내림차순 정렬**
- 가방마다:
  - 보석 리스트를 앞에서부터 탐색
  - 들어갈 수 있는 첫 보석 선택 후 제거

## 🚨 문제점

### 1. 시간복잡도 폭발
```java
for (가방) {
    for (보석 전체 탐색) {
        ...
    }
}
```

👉 **O(NK)** → 시간초과

---

### 2. ArrayList.remove(i)
```java
jewel.remove(i);
```

👉 중간 삭제 = **O(N)**  
👉 전체 성능 더 악화

---

### 3. 같은 보석 반복 검사
- 매 가방마다 보석 리스트 처음부터 다시 탐색
- 이미 고려한 보석을 계속 다시 봄

---

# ✅ 정석 풀이 (그리디 + 우선순위큐)

## 💡 핵심 아이디어
“현재 가방에 들어갈 수 있는 보석 중 가장 비싼 것 선택”

---

## 🔑 전략
1. **보석을 무게 오름차순 정렬**
2. **가방을 용량 오름차순 정렬**
3. 현재 가방에 들어갈 수 있는 보석을 모두 **힙에 추가**
4. 그 중 가장 비싼 보석 선택

---

## 📦 자료구조
- PriorityQueue<Integer> (최대힙) → 가격 기준

---

# ⚙️ 알고리즘 흐름
```java
int idx = 0;

for (각 가방) {

    while (idx < n && jewel[idx].weight <= bag[i]) {
        pq.offer(jewel[idx].price);
        idx++;
    }

    if (!pq.isEmpty()) {
        ans += pq.poll();
    }
}
```

---

# ❗ 중요 개념

## 1. 보석은 한 번만 사용됨

### 이유 1️⃣: idx는 뒤로 가지 않음
👉 보석은 한 번만 pq에 들어감

### 이유 2️⃣: poll() 하면 제거됨
👉 한 번 쓰면 pq에서 사라짐

---

# 📊 흐름 예시

보석:
(2,100), (2,90), (3,80)

가방:
2, 3

### 가방 2
- pq: [100, 90]
- 선택: 100

### 가방 3
- pq: [90, 80]
- 선택: 90

👉 같은 보석은 다시 사용되지 않음

---

# ⏱️ 시간복잡도

- 보석 정렬: O(N log N)
- 가방 정렬: O(K log K)
- 힙 연산: O((N+K) log N)

👉 최종: O((N+K) log N)

---

# ⚠️ 주의사항

## 1. 정답은 long
```java
long ans = 0;
```

## 2. 힙에는 가격만 넣어도 됨
👉 무게 조건 통과한 보석만 넣기 때문

---

# 🧠 패턴 정리

- 제한 있음 (가방)
- 아이템 1개 선택
- 가치 최대화
- 중복 사용 불가

👉 그리디 + 정렬 + 우선순위큐
