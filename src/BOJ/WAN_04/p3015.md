# 오답노트 - 오아시스 재결합 (BOJ)

## 문제 요약

N명이 한 줄로 서 있을 때, 두 사람 A와 B 사이에 A 또는 B보다 키가 큰 사람이 없으면 서로 볼 수 있다.  
서로 볼 수 있는 **쌍의 수**를 구하는 문제.

---

## 핵심 아이디어

**단조 감소 스택**을 유지하면서, 원소가 pop될 때 쌍의 수를 계산한다.

pop된 그룹(중복 개수 d)이 볼 수 있는 상대:

| 대상 | 쌍의 수 |
|---|---|
| 같은 값끼리 | `d*(d-1)/2` (조합) |
| 오른쪽 (pop을 유발한 arr[i]) | `d` |
| 왼쪽 (스택에 남은 top) | `d` (스택이 비어있으면 없음) |

루프가 끝난 후 스택에 남은 원소도 후처리 필요:

| 대상 | 쌍의 수 |
|---|---|
| 같은 값끼리 | `d*(d-1)/2` |
| 왼쪽 스택 top | `d` (스택이 비어있으면 없음) |

> 오른쪽 쌍은 없다. 스택에 남은 원소들은 더 큰 값이 오른쪽에 없어서 pop이 안 된 것들이기 때문.

---

## 틀렸던 이유들

### 1. `i - idx - 1`로 쌍을 계산

```java
int x = i - idx - 1;
cnt += x;  // ❌
```

이건 두 인덱스 사이의 **거리**이지 쌍의 수가 아니다.  
pop될 때 더해야 할 것은 오른쪽 1쌍, 왼쪽 1쌍 (그리고 중복 처리)이다.

### 2. 중복값 처리 누락

같은 키를 가진 d명은 서로 전부 볼 수 있으므로 `d*(d-1)/2` 쌍이 추가된다.  
이를 처리하지 않으면 같은 키끼리의 쌍이 전부 누락된다.

예) `[3, 3, 3]` → 쌍: (0,1), (1,2), (0,2) = **3쌍** = `3*2/2`

### 3. `stack.poll()` 혼용

```java
int[] x = stack.poll();  // ❌ 큐처럼 바닥에서 꺼냄
```

`push/peek/pop`으로 스택처럼 쓰다가 `poll`을 쓰면 꺼내는 순서가 반대가 된다.  
항상 `stack.pop()`을 사용해야 한다.

### 4. 루프를 `n-1`까지만 순회

```java
for(int i = 0; i < n-1; i++)  // ❌ 마지막 원소 누락
```

마지막 원소가 스택에 push조차 되지 않아 후처리와 불일치 발생.  
`i < n`으로 전체 순회해야 한다.

### 5. 스택 후처리 누락

루프가 끝난 후 스택에 남은 원소들의 쌍을 처리하지 않으면 결과가 틀린다.

### 6. 중복 처리 후 push 누락

```java
if (arr[stack.peek()[0]] == arr[i]) {
    idx += stack.pop()[1];
}
else stack.push(...);  // ❌ else라서 중복일 때 push가 안 됨
```

중복값일 때 pop만 하고 새로 push를 안 하면 스택에서 사라진다.  
중복이든 아니든 항상 `stack.push()`를 해야 한다.

### 7. d 초기값 오류

```java
int d = 1;
d += x[1];  // ❌ x[1]이 이미 그룹 개수인데 1을 더 더함
```

`x[1]` 자체가 중복 그룹의 개수이므로 `int d = x[1]`이어야 한다.

## 최종 정답 코드

```java
import java.io.*;
import java.util.*;

public class Main {
    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

        int n = Integer.parseInt(br.readLine());
        long cnt = 0;
        int[] arr = new int[n];
        Deque<int[]> stack = new ArrayDeque<>(); // {인덱스, 중복 수}

        for (int i = 0; i < n; i++) arr[i] = Integer.parseInt(br.readLine());

        for (int i = 0; i < n; i++) {
            int dup = 1;
            while (!stack.isEmpty() && arr[stack.peek()[0]] < arr[i]) {
                int[] x = stack.pop();
                int d = x[1];
                cnt += (long) d * (d - 1) / 2; // 중복끼리
                cnt += d;                        // 오른쪽 arr[i]와
                if (!stack.isEmpty()) cnt += d;  // 왼쪽 스택 top과
            }
            if (!stack.isEmpty() && arr[stack.peek()[0]] == arr[i]) {
                dup += stack.pop()[1];
            }
            stack.push(new int[]{i, dup}); // 항상 push
        }

        // 스택에 남은 원소 후처리
        while (!stack.isEmpty()) {
            int[] x = stack.pop();
            int d = x[1];
            cnt += (long) d * (d - 1) / 2;
            if (!stack.isEmpty()) cnt += d;
        }

        System.out.println(cnt);
    }
}
```

## 체크리스트

- [ ] 루프는 `i < n`으로 전체 순회
- [ ] pop할 때 `d = x[1]` (1 더하지 말 것)
- [ ] 오른쪽 쌍 `+d`, 왼쪽 쌍 `+d` (스택 비면 생략), 중복 쌍 `+d*(d-1)/2`
- [ ] 중복 처리 후 항상 `stack.push()`
- [ ] `stack.pop()` 사용 (`poll()` 금지)
- [ ] 루프 종료 후 스택 후처리
