# 🧠 게리맨더링(BOJ 17471) 오답노트

## 📌 문제 요약

* n개의 구역이 있음 (n ≤ 10)
* 각 구역에는 인구가 있음
* 구역을 두 개의 선거구로 나눠야 함
* 각 선거구는 **연결되어 있어야 함**
* 두 선거구의 인구 차이를 최소화

---

# ❌ 내가 했던 실수

## 1. DFS로 "연결된 집합"을 직접 만들려고 함

초기 접근:

```
DFS로 연결된 정점들만 선택해서 집합을 만들자
```

### 👉 왜 틀렸나

이 방식은 생각보다 구현이 매우 까다롭다.

문제 상황:

* 현재 선택 집합이 {1, 2}일 때
* 다음에 추가할 수 있는 후보는

  * "마지막 정점"의 이웃이 아니라
  * "집합 전체와 연결된 정점"이어야 함

하지만 코드에서는:

```
마지막 정점 v의 인접 노드만 탐색
```

→ 결과: 일부 가능한 경우를 **아예 생성 못함**

---

## 2. 생성과 검사를 섞어서 생각함

잘못된 사고:

```
조건을 만족하는 경우만 DFS로 만들자
```

문제점:

* 조건이 복잡해질수록 DFS 설계가 어려워짐
* 빠지는 경우 발생 가능

---

# ✅ 올바른 접근

## 핵심 아이디어

```
생성과 검사를 분리한다
```

---

## ✔ Step 1: 부분집합 생성

* 모든 구역을 대상으로
* 하나의 선거구(A)를 선택

방법:

* 조합 / 부분집합 DFS
* 또는 비트마스킹

---

## ✔ Step 2: 연결 여부 검사

각 경우마다

* A 그룹 연결 여부 검사
* B 그룹(보완집합) 연결 여부 검사

방법:

* BFS 또는 DFS

---

## ✔ Step 3: 인구 차 계산

```
|total - 2 * sum(A)|
```

---

# 🔥 핵심 교훈

## ❗ 연결 조건이 있다고 해서

## ❗ DFS로 직접 만들려고 하지 말 것

---

## 올바른 사고 흐름

```
1. 경우의 수가 작다 → 전부 생성한다
2. 조건이 있다 → 생성 후 검사한다
```

---

# ⚠️ 헷갈리는 포인트

## ❌ 잘못된 접근

* "연결된 집합만 만들자"
* "DFS로 조건 만족하게 구성하자"

👉 구현 난이도 상승 + 누락 발생

---

## ✅ 정석 접근

* "일단 다 만들어"
* "그 다음 조건 체크"

👉 단순 + 안정적

---

# 🧠 문제 유형 정리

## 👉 유형

```
완전탐색 + 그래프 연결 판정
```

---

## 👉 구조

```
부분집합 선택
→ 각 집합 연결 확인
→ 값 계산
```

---

# 🚀 실전 판단 기준

문제에서 이런 키워드가 보이면:

* "두 그룹으로 나눈다"
* "조건을 만족하는 집합"
* "n ≤ 10"

👉 바로 떠올려야 함:

```
완전탐색 + 조건 검사
```

---

# 🏁 한 줄 정리

```
연결 조건은 DFS로 '만들' 게 아니라
DFS/BFS로 '검사'하는 것이다
```
