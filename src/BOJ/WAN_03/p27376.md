# 🚦 BOJ 27376 오답노트 (신호등)

## ❌ 내가 틀린 이유

### 1. 빨간불 대기 시간 계산 오류

초기 코드에서 다음 초록 신호 시작 시점을 잘못 계산함.

```java
int y = s + (time - s) / t;
time = y * t + s;
```

👉 문제점:

* `s`를 기준으로 한 **주기 계산이 잘못됨**
* `(time - s) / t`는 “현재 구간 번호”인데,
  이를 그대로 사용하면 안 됨

---

### ✔️ 올바른 계산

```java
long q = (time - s) / t;
time = s + (q + 1) * t;
```

또는

```java
long diff = time - s;
time += t - (diff % t);
```

---

### 2. 시작 기준 (time, dist) 잘못 설정

```java
int time = 1;
int dist = 1;
```

👉 문제점:

* 이미 1초 이동한 상태로 시작하는 꼴
* 신호등 도착 시점이 어긋남

---

### ✔️ 올바른 시작

```java
long time = 0;
int dist = 0;
```

---

### 3. 시뮬레이션 방식 자체의 문제

기존 방식:

* 1칸씩 이동하면서 처리
* `dist == 신호등 위치`일 때 체크

👉 문제점:

* 도착 시점 vs 이동 시점이 섞여서 헷갈림
* 시간 계산이 꼬이기 쉬움

---

## ✅ 정석 풀이 방식

### 핵심 아이디어

👉 **신호등 위치로 한 번에 이동 (점프)**

1. 현재 위치 → 다음 신호등까지 이동
2. 도착 시간 계산
3. 신호 상태 확인 후 대기
4. 다음으로 이동

---

### 코드 구조

```java
time += x - pos;

if (time < s) {
    time = s;
} else {
    long q = (time - s) / t;
    if ((q & 1) == 1) {
        time = s + (q + 1) * t;
    }
}
```

---

## 🧠 핵심 개념 정리

### 🔹 신호등 주기

* `s`: 첫 초록 시작 시간
* `t`: 주기
* `(time - s) / t`:
  → 현재가 몇 번째 구간인지

| 값  | 의미 |
| -- | -- |
| 짝수 | 초록 |
| 홀수 | 빨강 |

---

### 🔹 빨간불일 때 처리

```java
if ((q & 1) == 1)
```

👉 다음 초록 시작 시점으로 이동

```java
time = s + (q + 1) * t;
```

---

### 🔹 시간 처리 주의

* 항상 `long` 사용
* 누적되기 때문에 overflow 위험 있음

## ⚠️ 실전에서의 교훈

### 1. “이벤트 지점 기준으로 이동하라”

* 1칸씩 시뮬레이션 ❌
* 중요한 지점(신호등) 기준으로 점프 ⭕

---

### 2. “시간 기준 문제는 도착 시점 기준으로 생각”

* 이동 중 ❌
* 도착 순간 ⭕

---

### 3. “주기 문제는 기준점(s)을 절대 잊지 말 것”

* `(time - s)` 형태 필수

---

## 🚀 한 줄 요약

👉 **신호등 문제는 “도착 시간 기준 + 주기 계산 + 다음 이벤트로 점프”가 핵심이다.**
