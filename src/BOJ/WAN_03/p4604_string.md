# 🧠 Steganography 문제 풀이 정리 (README)

## 📌 문제 핵심

문장 내 **연속된 공백 개수**를 이용해 숨겨진 비트를 해독하는 문제.

------------------------------------------------------------------------

## 🔑 규칙 정리

### 1. 공백 → 비트 변환

-   공백 개수 **홀수** → `0`
-   공백 개수 **짝수** → `1`

------------------------------------------------------------------------

### 2. 비트 → 문자 변환 (5비트 단위)

  값      문자
  ------- ---------
  0       (space)
  1\~26   A\~Z
  27      '
  28      ,
  29      \-
  30      .
  31      ?

------------------------------------------------------------------------

### 3. 예외 처리

-   마지막 비트가 5개 미만이면 → **오른쪽에 0 padding**

------------------------------------------------------------------------

## 🚀 풀이 전략 (정석)

1.  입력을 한 줄씩 읽는다
2.  `*`가 나오면 하나의 메시지 종료
3.  공백 묶음을 발견할 때마다:
    -   길이 계산
    -   비트로 변환
4.  5비트가 모이면 바로 문자로 변환

------------------------------------------------------------------------

## 💻 핵심 코드

``` java
int value = 0;
int bits = 0;

int bit = (cnt % 2 == 0) ? 1 : 0;
value = (value << 1) | bit;
bits++;

if (bits == 5) {
    출력
    value = 0;
    bits = 0;
}
```

------------------------------------------------------------------------

## ⚠️ 자주 틀리는 포인트

### 1. 비트 반대로 넣는 실수

❌ `(cnt & 1)`\
→ 홀수 = 1 (틀림)

✅ `(cnt % 2 == 0 ? 1 : 0)`

------------------------------------------------------------------------

### 2. 공백 개수 잘못 세기

❌ `cnt = 0` 시작\
→ 첫 공백 빠짐

✅ `cnt = 1` 시작

------------------------------------------------------------------------

### 3. 문자 완성 후 초기화 안 함

``` java
value = 0;
bits = 0;
```

------------------------------------------------------------------------

### 4. `*` 처리 실수

❌ 항상 문자 하나 더 출력\
✅ `bits > 0`일 때만 처리

------------------------------------------------------------------------

## ⏱️ 시간복잡도

-   O(N) (문자 한 번씩만 순회)

------------------------------------------------------------------------

## 🧾 한 줄 요약

👉 **공백 묶음 → 비트 → 5비트 → 문자** 흐름을 그대로 구현하면 된다.
