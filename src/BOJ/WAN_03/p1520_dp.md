# 📝 알고리즘 오답노트

## 문제 정보

| 항목 | 내용 |
|------|------|
| 유형 | 그래프 탐색 (DFS + DP) |
| 핵심 개념 | 메모이제이션, DAG, 백트래킹 |

---

## ❌ 틀린 코드

```java
static void dfs(int r, int c) {
    if (r == n-1 && c == m-1) { cnt++; return; }
    if (block[r][c] || visited[r][c]) return;
    visited[r][c] = true;
    for (int i = 0; i < 4; i++) {
        int nr = r + dr[i];
        int nc = c + dc[i];
        if (nr < 0 || nc < 0 || nr >= n || nc >= m) continue;
        if (map[r][c] <= map[nr][nc]) continue;
        dfs(nr, nc);
    }
    visited[r][c] = false;
}
```

### 문제점

- `visited` 배열로 백트래킹하며 **모든 경로를 매번 재탐색**
- 같은 `(r, c)`에 도달하는 경로가 여러 개일 경우, 해당 지점에서의 탐색을 **중복으로 반복**
- 시간복잡도: **O(4^(n×m))** — 지수적으로 증가

---

## ✅ 맞는 코드

```java
static int dfs(int r, int c) {
    if (r == n-1 && c == m-1) return 1;
    if (memo[r][c] != -1) return memo[r][c];  // 캐시 히트

    memo[r][c] = 0;
    for (int i = 0; i < 4; i++) {
        int nr = r + dr[i];
        int nc = c + dc[i];
        if (nr < 0 || nc < 0 || nr >= n || nc >= m) continue;
        if (map[r][c] <= map[nr][nc]) continue;
        memo[r][c] += dfs(nr, nc);
    }
    return memo[r][c];
}
```

---

## 💡 핵심 인사이트

### 왜 `visited`가 필요 없는가?

이 문제는 **내리막 조건**이 존재한다:

```java
if (map[r][c] <= map[nr][nc]) continue;  // 현재보다 낮은 곳만 이동
```

이동할 때마다 값이 **반드시 감소**하기 때문에 이미 방문한 곳으로 돌아오는 것이 불가능하다.

```
10 → 8 → 5 → 3 → 1  (값이 줄어들기만 함)
```

즉, 그래프 구조 자체가 **DAG (Directed Acyclic Graph, 방향 비순환 그래프)** 이다.

### DAG에서 메모이제이션이 가능한 이유

`(r, c)`에서 목적지까지의 경로 수는, **어떤 경로로 (r, c)에 도달했는지와 무관하게 항상 동일**하다.  
다음으로 갈 수 있는 칸은 오직 `map` 값에 의해서만 결정되기 때문이다.

따라서 한 번 계산한 결과를 `memo`에 저장해두면 이후 동일한 칸에 도달했을 때 즉시 반환할 수 있다.

## 📊 성능 비교

| 항목 | 기존 (백트래킹) | 개선 (메모이제이션) |
|------|---------------|-----------------|
| 시간복잡도 | O(4^(n×m)) | **O(n×m)** |
| `(r,c)` 재방문 | 매번 재탐색 | 캐시에서 즉시 반환 |
| `visited` 배열 | 필요 (사이클 방지 착각) | **불필요** |
| 실제 계산 횟수 | 지수적 | 각 셀 최대 1번 |

---

## 🔑 기억할 것

> **이동 조건에 단조성(값이 항상 감소/증가)이 있으면 → 사이클 없음 → DAG → 메모이제이션 가능**

- 단순히 `visited`로 중복 방문을 막는 것과, `memo`로 결과값을 캐싱하는 것은 **완전히 다른 개념**
- DAG 구조를 파악했다면 백트래킹 없이 DP로 풀어야 한다는 신호
- `memo` 배열 초기값은 `-1` (미방문)로 설정해 `0`인 경우(경로 없음)와 구분할 것
