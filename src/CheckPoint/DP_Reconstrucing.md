# 역추적 방식 정리

## 1. 역추적이란?

역추적은 **DP나 탐색으로 구한 최적값으로부터 실제 선택 경로를 복원하는 방법**이다.

예를 들어:

* 최장 증가 부분 수열의 길이만 구하는 것
* LCS의 길이만 구하는 것
* 최단 거리의 값만 구하는 것

여기서 한 단계 더 나아가,

* 실제 LIS 수열
* 실제 LCS 문자열
* 실제 최단 경로

를 구하려면 역추적이 필요하다.

즉,

* **값 계산 = 최적값 구하기**
* **역추적 = 그 최적값이 만들어진 과정 복원하기**

로 생각하면 된다.

---

## 2. 역추적 방식은 왜 여러 종류가 있나?

문제마다 상태 정의가 다르기 때문이다.

예를 들어:

* 상태가 1차원인 문제
* 상태가 2차원인 문제
* 이전 상태가 하나로 정해지는 문제
* 여러 방향 중 하나를 선택해야 하는 문제

이런 차이 때문에 복원 방식도 달라진다.

대표적으로 아래 4가지로 정리할 수 있다.

1. `prev` 방식
2. `how` / 방향 저장 방식
3. `dp 테이블만 보고 역추적` 방식
4. 탐색 과정의 부모 저장 방식

---

## 3. prev 방식

## 개념

현재 상태가 **어떤 이전 상태에서 왔는지 인덱스 하나로 저장**하는 방식이다.

보통 상태가 1차원일 때 잘 맞는다.

예:

* LIS 복원
* 최단 경로에서 이전 정점 저장
* 선택된 이전 위치가 하나로 결정되는 DP

## 형태

```java
int[] prev = new int[n];
Arrays.fill(prev, -1);
```

의미:

* `prev[i] = i 상태 직전에 온 상태`

## 예시: LIS 복원

```java
for (int i = 0; i < n; i++) {
    dp[i] = 1;
    prev[i] = -1;
    for (int j = 0; j < i; j++) {
        if (arr[j] < arr[i] && dp[i] < dp[j] + 1) {
            dp[i] = dp[j] + 1;
            prev[i] = j;
        }
    }
}
```

복원:

```java
List<Integer> ans = new ArrayList<>();
int cur = lastIndex;
while (cur != -1) {
    ans.add(arr[cur]);
    cur = prev[cur];
}
Collections.reverse(ans);
```

## 장점

* 구현이 쉽다
* 메모리 사용이 적다
* 상태가 1차원일 때 매우 직관적이다

## 주의점

* **현재 상태의 이전 상태가 하나로 정해질 때** 잘 맞는다
* 상태가 2차원 이상이거나 경로 선택이 복잡하면 부적합할 수 있다

## 4. how / 방향 저장 방식

## 개념

각 상태가 **어디서 왔는지 방향이나 종류를 저장**하는 방식이다.

예를 들어:

* 위에서 왔는지
* 왼쪽에서 왔는지
* 대각선에서 왔는지

같은 정보를 저장한다.

보통 2차원 DP에서 자주 사용한다.

## 형태

```java
char[][] how = new char[n + 1][m + 1];
```

예:

* `'U'` : 위에서 옴
* `'L'` : 왼쪽에서 옴
* `'D'` : 대각선에서 옴

## 예시: LCS

```java
for (int i = 1; i <= n; i++) {
    for (int j = 1; j <= m; j++) {
        if (a.charAt(i - 1) == b.charAt(j - 1)) {
            dp[i][j] = dp[i - 1][j - 1] + 1;
            how[i][j] = 'D';
        } else if (dp[i - 1][j] >= dp[i][j - 1]) {
            dp[i][j] = dp[i - 1][j];
            how[i][j] = 'U';
        } else {
            dp[i][j] = dp[i][j - 1];
            how[i][j] = 'L';
        }
    }
}
```

복원:

```java
StringBuilder sb = new StringBuilder();
int i = n, j = m;
while (i > 0 && j > 0) {
    if (how[i][j] == 'D') {
        sb.append(a.charAt(i - 1));
        i--;
        j--;
    } else if (how[i][j] == 'U') {
        i--;
    } else {
        j--;
    }
}
System.out.println(sb.reverse().toString());
```

## 장점

* 복원 과정이 명확하다
* DP를 계산하면서 동시에 경로를 기록할 수 있다
* 디버깅이 쉬운 편이다

## 주의점

* 방향 저장 배열이 추가로 필요하다
* 같은 최적값이 여러 개일 때 어떤 방향을 고를지 기준이 필요하다

---

## 5. dp 테이블만 보고 역추적하는 방식

## 개념

별도의 `how` 배열 없이, **이미 계산된 dp 값들만 비교하면서 거꾸로 이동**하는 방식이다.

LCS가 대표적이다.

## 예시: LCS

```java
while (i > 0 && j > 0) {
    if (a.charAt(i - 1) == b.charAt(j - 1)) {
        sb.append(a.charAt(i - 1));
        i--;
        j--;
    } else if (dp[i - 1][j] >= dp[i][j - 1]) {
        i--;
    } else {
        j--;
    }
}
```

## 왜 가능한가?

`dp[i][j]`의 값이

* 위에서 왔는지
* 왼쪽에서 왔는지
* 대각선에서 왔는지

를 `dp` 값 비교로 유추할 수 있기 때문이다.

## 장점

* `how` 배열이 필요 없다
* 메모리를 조금 아낄 수 있다
* 구현이 간단하다

## 주의점

* 문제에 따라 값만 보고는 이전 상태를 확정할 수 없는 경우도 있다
* 여러 최적 경로가 있으면 복원 결과가 여러 개 중 하나가 될 수 있다
* 점화식을 정확히 이해하고 있어야 한다

---

## 6. 탐색 과정의 부모 저장 방식

## 개념

BFS, DFS, 다익스트라 같은 그래프 탐색에서 **현재 정점에 오기 직전의 정점**을 저장하는 방식이다.

이것도 사실상 `prev`의 확장판이다.

## 형태

```java
int[] parent = new int[n + 1];
Arrays.fill(parent, -1);
```

## 예시: BFS 최단 경로 복원

```java
Queue<Integer> q = new LinkedList<>();
q.add(start);
visited[start] = true;
parent[start] = -1;

while (!q.isEmpty()) {
    int cur = q.poll();
    for (int next : graph[cur]) {
        if (!visited[next]) {
            visited[next] = true;
            parent[next] = cur;
            q.add(next);
        }
    }
}
```

복원:

```java
List<Integer> path = new ArrayList<>();
int cur = end;
while (cur != -1) {
    path.add(cur);
    cur = parent[cur];
}
Collections.reverse(path);
```

## 장점

* 최단 경로 문제에서 매우 자주 쓰인다
* 구현이 직관적이다
* 정점 경로 복원에 적합하다

## 주의점

* 방문 처리를 언제 하느냐에 따라 부모가 달라질 수 있다
* 다익스트라처럼 더 좋은 경로로 갱신될 수 있는 문제는 parent도 함께 갱신해야 한다

## 7. 선택 여부를 저장하는 방식

## 개념

배낭 문제처럼 “이 물건을 선택했는가?”가 중요한 경우,
각 상태에서 **선택했는지 안 했는지**를 저장해서 복원한다.

## 예시 개념

```java
boolean[][] take = new boolean[n + 1][W + 1];
```

* `take[i][w] = true` 이면
  `i번째 물건을 사용해서 dp[i][w]가 만들어졌다는 뜻`

복원은 뒤에서부터:

* 선택했으면 물건 포함
* 아니면 넘어감

## 장점

* 0/1 배낭처럼 선택 문제가 명확할 때 좋다

## 주의점

* 상태 수가 많으면 메모리 사용량이 커진다

## 8. 역추적 방식 선택 기준

## (1) 상태가 1차원이고 이전 상태가 하나면

`prev` 방식이 좋다.

예:

* LIS
* 선형 DP 일부
* 그래프 parent 저장

## (2) 상태가 2차원이고 이동 방향이 중요하면

`how[i][j]` 또는 `dp 테이블 역추적`이 좋다.

예:

* LCS
* 문자열 DP
* 격자 DP

## (3) 선택 문제면

`take` 같은 선택 여부 저장이 좋다.

예:

* 배낭 문제
* 부분집합 선택 문제

## (4) 그래프 최단 경로면

`parent` 저장이 좋다.

예:

* BFS
* 다익스트라

## 9. LCS에서 왜 `how[len]`가 안 되는가?

이 부분은 특히 중요하다.

예를 들어 LCS는 상태가 `dp[i][j]`다.

즉,

* `x의 앞 i개`
* `y의 앞 j개`

를 함께 고려한 결과다.

그런데 `how[len] = 어떤 문자 위치`처럼 저장하면,

* 길이 1을 만든 위치
* 길이 2를 만든 위치
* 길이 3을 만든 위치

가 **같은 경로 위의 선택인지 보장되지 않는다.**

그래서 LCS는

* `how[i][j]`처럼 상태별로 방향을 저장하거나
* `dp[i][j]` 값 자체를 보고 역추적해야 한다.

즉,

* **LIS는 마지막 원소 기반 복원이 쉬운 문제**
* **LCS는 상태가 2차원이라 경로 자체를 따라가야 하는 문제**

라고 보면 된다.

## 10. 실전에서 가장 많이 쓰는 복원 템플릿

## 템플릿 A: prev

```java
int cur = end;
while (cur != -1) {
    ans.add(cur);
    cur = prev[cur];
}
Collections.reverse(ans);
```

## 템플릿 B: 2차원 how

```java
while (i > 0 && j > 0) {
    if (how[i][j] == 'D') {
        // 문자 사용
        i--;
        j--;
    } else if (how[i][j] == 'U') {
        i--;
    } else {
        j--;
    }
}
```

## 템플릿 C: dp 값 비교 역추적

```java
while (i > 0 && j > 0) {
    if (match) {
        // 사용
        i--;
        j--;
    } else if (dp[i - 1][j] >= dp[i][j - 1]) {
        i--;
    } else {
        j--;
    }
}
```

## 11. 공부할 때 기억할 핵심

1. 먼저 **dp 상태 정의**를 정확히 써라.
2. 그 상태가 **어디서 왔는지**를 생각해라.
3. 이전 상태가 하나면 `prev`가 잘 맞는다.
4. 이전 상태가 여러 방향이면 `how` 또는 `dp 역추적`이 필요하다.
5. 복원 문제는 결국 **최적값이 아니라 경로를 저장하는 문제**다.

---

## 12. 한 줄 정리

* **1차원 상태 복원**: `prev`
* **2차원 문자열/격자 복원**: `how[][]` 또는 `dp 역추적`
* **그래프 경로 복원**: `parent`
* **선택 문제 복원**: `take[][]`

즉, 역추적은 하나의 고정된 방법이 아니라,
**상태 정의에 맞는 부모 정보 저장 방식**이라고 이해하면 된다.
