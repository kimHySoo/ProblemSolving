# 🧠 치킨 배달 문제 유형 판단 정리 (코테 필수 감각)

## 📌 문제 상황

문제를 처음 봤을 때

* 다익스트라?
* MST?
* 브루트포스?

이렇게 헷갈릴 수 있다.

👉 이 README는 **이 문제를 한 번에 브루트포스로 판단하는 방법**을 정리한 것이다.

---

# 🔥 핵심 결론

```
선택 + 최소/최대 → 브루트포스 (조합/백트래킹)
```

---

# 1️⃣ 가장 중요한 신호: "선택"

문제에서 가장 중요한 문장:

> 치킨집 중 M개를 선택

이 문장을 보면 바로 떠올려야 한다.

```
선택이 있다 → 조합 문제
```

### ✔ 의미

* 어떤 원소를 고르냐에 따라 결과가 달라짐
* 모든 경우를 확인해야 함

👉 따라서 **브루트포스 / 백트래킹**으로 이어진다.

---

# 2️⃣ 그래프 문제인지 확인하는 법

## ❌ 이 문제는 그래프가 아니다

그래프 문제라면 보통:

* 간선 존재
* 노드 간 연결
* 경로를 따라 이동

하지만 이 문제는:

* 단순 거리 계산 (맨해튼 거리)
* 경로 없음
* 중간 노드 없음

👉 따라서

```
다익스트라 ❌
MST ❌
```

---

# 3️⃣ 그래프 알고리즘이 나오는 조건

## ✔ 다익스트라

* 시작점 존재
* 간선을 따라 이동
* 최단 경로 문제

## ✔ MST

* 모든 노드 연결
* 간선 선택
* 사이클 제거

---

## ❗ 이 문제와 비교

| 요소 | 존재 여부 |
| -- | ----- |
| 경로 | ❌     |
| 간선 | ❌     |
| 이동 | ❌     |
| 연결 | ❌     |

👉 그래프 아님 → 바로 제외

---

# 4️⃣ 문제 구조 분석

문제 핵심:

1. 치킨집 M개 선택
2. 각 집에서 가장 가까운 치킨집 거리 계산
3. 총합 최소화

👉 즉,

```
선택된 집합에 따라 결과가 달라짐
```

이건 매우 중요한 포인트:

👉 **"정답이 선택에 의존한다" → 브루트포스**

---

# 5️⃣ 입력 크기로 확신하기

치킨집 최대 개수 ≈ 13

가능한 경우:

* 2^13 = 8192
* C(13, M)도 충분히 작음

👉 즉

```
브루트포스 가능
```

---

# 🧠 실전 판단 루틴 (암기 추천)

문제를 보면 다음 순서로 판단:

### 1️⃣ 선택이 있는가?

* YES → 조합/백트래킹

### 2️⃣ 경로/그래프 구조가 있는가?

* NO → 다익스트라/MST 제외

### 3️⃣ 입력 크기가 작은가?

* YES → 브루트포스 확정

---

# ⚠️ 헷갈리는 이유

많은 사람들이 착각하는 이유:

👉 "거리 최소"라는 키워드 때문

하지만 구분해야 한다:

| 유형    | 의미                |
| ----- | ----------------- |
| 다익스트라 | 한 점 → 다른 점까지 최단거리 |
| MST   | 전체 연결 최소          |
| 이 문제  | 선택된 집합 기준 거리      |

👉 완전히 다른 문제이다.

---

# 🔥 핵심 한 줄 정리

```
이 문제는 "경로 최적화"가 아니라
"선택 후 거리 계산"이다.
```

---

# 🚀 유형 정리

이 문제는 다음 유형이다:

## 👉 조합 + 시뮬레이션

### 대표 문제들

* 치킨 배달
* 연구소 (벽 세우기)
* 감시 (CCTV)

### 공통 구조

```
1. 선택한다
2. 결과를 계산한다
```

---

# 💡 마무리

이 문제를 빠르게 판단하는 핵심은:

* "선택"이라는 키워드 캐치
* 그래프 구조 여부 확인
* 입력 크기 체크

👉 이 3개만 해도 대부분 문제 유형을 빠르게 구분할 수 있다.
