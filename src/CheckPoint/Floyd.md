# 플로이드-워셜에서 `k`의 의미와 경로 순서가 다른 이유

## 1. 왜 헷갈리는가

플로이드-워셜을 처음 배우면 이런 의문이 자주 듭니다.

* 반복문은 `k = 1`부터 차례대로 돈다.
* 그런데 실제 최단 경로는 `2 → 4 → 5 → 1 → 3`처럼 생길 수 있다.
* 그러면 `k=1`일 때 `2 → 1 → 3`을 봐야 하는데, 아직 `2 → 1`은 `2 → 4 → 5 → 1`로 계산되지 않았다.
* 그렇다면 최단 경로가 누락되는 것 아닌가?

이 의문은 아주 자연스럽습니다.
핵심은 **플로이드-워셜의 `k`는 실제 경로를 따라가는 순서가 아니라, "중간 정점으로 사용할 수 있는 정점의 범위"를 의미한다**는 점입니다.

---

## 2. `k`의 진짜 의미

플로이드-워셜은 다음 의미를 가진 DP입니다.

`dp[k][i][j] = 1번 ~ k번 정점만 중간 정점으로 사용할 수 있을 때 i → j 최단거리`

즉,

* `k=0` : 중간 정점을 하나도 사용하지 않음
* `k=1` : 1번 정점만 중간에 사용할 수 있음
* `k=2` : 1번, 2번 정점까지 중간에 사용할 수 있음
* ...
* `k=n` : 모든 정점을 중간에 사용할 수 있음

여기서 중요한 건 **`k`가 실제 이동 순서가 아니라 허용되는 중간 정점 집합의 크기**라는 것입니다.

---

## 3. 점화식의 의미

점화식은 다음과 같습니다.

```text
dp[k][i][j] = min(dp[k-1][i][j], dp[k-1][i][k] + dp[k-1][k][j])
```

의미는 다음과 같습니다.

* `dp[k-1][i][j]`

  * 정점 `k`를 전혀 사용하지 않고 가는 기존 최단거리
* `dp[k-1][i][k] + dp[k-1][k][j]`

  * 정점 `k`를 중간에 한 번 거쳐서 가는 경우

즉,

**"i에서 j로 갈 때, k를 거치는 것이 더 유리한가?"**

를 매 단계 검사하는 것입니다.

실제 구현에서는 3차원 배열 대신 2차원 배열 하나로 덮어쓰며 다음처럼 씁니다.

```java
for (int k = 1; k <= n; k++) {
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= n; j++) {
            dist[i][j] = Math.min(dist[i][j], dist[i][k] + dist[k][j]);
        }
    }
}
```

---

## 4. 실제 경로 순서와 `k` 순서는 다르다

예를 들어 실제 최단 경로가 다음과 같다고 합시다.

```text
2 → 4 → 5 → 1 → 3
```

이 경로의 중간 정점은 `4, 5, 1`입니다.

그런데 플로이드-워셜은 `k=1,2,3,4,5` 순서로 진행합니다.
처음 보면 실제 경로 순서와 완전히 다르므로 이상해 보입니다.

하지만 플로이드-워셜은

* `2 → 4`
* `4 → 5`
* `5 → 1`
* `1 → 3`

을 실제 경로 순서대로 한 칸씩 이어 붙이는 알고리즘이 아닙니다.

대신,

**"중간 정점으로 1번까지 허용했을 때의 최단거리"**
**"중간 정점으로 2번까지 허용했을 때의 최단거리"**
**"중간 정점으로 3번까지 허용했을 때의 최단거리"**

를 차례대로 완성해 가는 DP입니다.

즉, **경로를 따라가는 방식이 아니라 가능한 중간 정점의 범위를 확장하는 방식**입니다.

---

## 5. 왜 `k=1`에서 바로 안 잡혀도 괜찮은가

질문을 그대로 써 보면:

> `k=1`일 때 `2 → 1 → 3`을 보려 해도, 아직 `2 → 1`이 `2 → 4 → 5 → 1`로 계산 안 된 상태 아닌가?

맞습니다.
**그 시점에는 아직 반영되지 않았을 수 있습니다.**
하지만 괜찮습니다.

이유는 플로이드-워셜이 **한 번의 `k`에서 최종 답을 만들려고 하지 않기 때문**입니다.

예를 들어:

* `k=1`에서는 중간 정점으로 `1`만 허용
* `k=4`에서는 중간 정점으로 `1,2,3,4` 허용
* `k=5`에서는 중간 정점으로 `1,2,3,4,5` 허용

이렇게 허용 범위가 점점 커집니다.

따라서 처음에는 `2 → 1`이 아직 충분히 짧게 계산되지 않았더라도,
나중에 `4`, `5`까지 허용되면 `2 → 4 → 5 → 1`이 `dist[2][1]`에 반영될 수 있습니다.

그 후 그 값이 다시 다른 정점과 결합되면서 `dist[2][3]`에도 최종적으로 반영됩니다.

즉,

**처음 단계에서 안 잡혀도, 뒤 단계에서 필요한 부분 경로가 먼저 계산되고 그 다음 전체 경로가 완성됩니다.**

---

## 6. 핵심 오해: `dist[i][k]`는 직접 간선만 뜻하지 않는다

많이 헷갈리는 포인트가 이것입니다.

`dist[i][k]`

를 보면 많은 사람이 이것을 그냥 `i → k` 직접 간선이라고 생각합니다.
하지만 플로이드-워셜에서 `k`번째 반복 중의 `dist[i][k]`는 그런 단순한 값이 아닙니다.

그 값은 이미 이전 단계들을 거치며,

**중간 정점으로 `1 ~ k-1`번까지 사용한 최단거리**

가 저장된 결과입니다.

즉,

* `dist[2][5]`는 단순히 직접 간선 `2 → 5`만 의미하는 것이 아니라
* 필요하면 `2 → 4 → 5` 같은 경로가 이미 반영된 값일 수 있습니다.

마찬가지로,

* `dist[5][3]`도 단순히 직접 간선이 아니라
* `5 → 1 → 3` 같은 경로가 반영되어 있을 수 있습니다.

그래서 어느 시점에는

```text
dist[2][3] = min(dist[2][3], dist[2][5] + dist[5][3])
```

같은 식으로 긴 경로 정보가 합쳐지며 최단거리가 완성됩니다.

---

## 7. `2 → 4 → 5 → 1 → 3` 예시를 개념적으로 따라가 보기

실제로는 모든 `i, j`를 동시에 갱신하지만, 이해를 위해 특정 경로만 따라가 보면 다음과 같습니다.

### 초기 상태

초기 `dist`는 보통 다음을 의미합니다.

* 자기 자신으로 가는 비용: `0`
* 직접 연결된 간선의 비용: 간선 가중치
* 연결되지 않은 경우: 매우 큰 값(INF)

이때는 아직 `2 → 1`이 직접 연결되어 있지 않다면 `INF`일 수 있습니다.

### `k=1`

이 단계에서는 정점 `1`만 중간에 둘 수 있습니다.

따라서

```text
dist[2][3] = min(dist[2][3], dist[2][1] + dist[1][3])
```

를 볼 수는 있지만, 아직 `dist[2][1]`이 충분히 작지 않다면 갱신이 안 됩니다.

이것은 정상입니다.

### `k=4`

이제 정점 `4`를 중간 정점으로 사용할 수 있으므로,
예를 들어 `2 → 4 → 5` 쪽의 일부 정보가 반영되기 시작할 수 있습니다.

### `k=5`

이제 정점 `5`도 중간 정점으로 허용되므로,
`2 → 4 → 5 → 1` 같은 정보가 `dist[2][1]`에 반영될 가능성이 생깁니다.

### 최종적으로

이런 식으로 필요한 부분 경로들이 먼저 짧아지고,
그 결과가 이후 다른 경로 갱신에 사용되면서 결국

```text
2 → 4 → 5 → 1 → 3
```

같은 복잡한 경로도 최단거리로 반영됩니다.

즉, 플로이드-워셜은

* 긴 경로를 한 번에 찾는 것이 아니라
* 짧은 부분 최단거리들을 축적하고
* 그것들을 조합하면서 전체 최단거리를 만드는 방식

입니다.

## 8. 왜 덮어쓰기(2차원 배열)로도 되는가

플로이드-워셜은 원래 3차원 DP 정의로 이해할 수 있지만, 실제 구현은 2차원 배열 하나만 씁니다.

그 이유는 `k`번째 단계에서 필요한 값들이

* `dist[i][j]`
* `dist[i][k]`
* `dist[k][j]`

이고, 이 값들이 현재 단계에서 사용되어도 논리적으로 문제가 없기 때문입니다.

즉, 바깥쪽 반복문을 반드시 `k`로 두면

```java
for (int k = 1; k <= n; k++)
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= n; j++)
            dist[i][j] = Math.min(dist[i][j], dist[i][k] + dist[k][j]);
```

이 형태로 `dp[k-1] → dp[k]` 전이가 올바르게 이루어집니다.

반대로 반복문 순서를 잘못 바꾸면 플로이드-워셜의 의미가 깨질 수 있습니다.

예를 들어 `i`, `j`를 바깥에 두고 `k`를 안쪽에 두면,
아직 허용되지 않아야 할 중간 정점 정보가 섞여 잘못된 결과가 나올 수 있습니다.

그래서 **플로이드-워셜은 `k`가 가장 바깥 반복문이어야 합니다.**

---

## 9. 이걸 DP 관점으로 다시 정리하면

플로이드-워셜은 브루트포스로 모든 경로를 직접 탐색하는 것이 아니라,
**"중간 정점 허용 범위를 점점 넓혀 가며 부분 문제를 해결하는 DP"**입니다.

부분 문제는 다음과 같습니다.

* 중간 정점을 하나도 쓰지 않고 가는 최단거리
* 1번까지 허용하고 가는 최단거리
* 2번까지 허용하고 가는 최단거리
* ...
* n번까지 허용하고 가는 최단거리

즉,

* 작은 문제: 허용되는 중간 정점 수가 적을 때의 최단거리
* 큰 문제: 허용되는 중간 정점 수가 많을 때의 최단거리

로 확장해 가는 전형적인 DP입니다.

## 10. 자주 하는 오해 정리

### 오해 1. `k`는 실제 경로에서 몇 번째로 방문하는 정점이다

아닙니다.
`k`는 **실제 방문 순서**가 아니라 **중간 정점으로 사용할 수 있게 허용한 정점 번호 범위**입니다.

### 오해 2. `k=1`일 때 최종 경로가 바로 반영되어야 한다

아닙니다.
`k=1`은 단지 **1번 정점까지만 중간에 사용할 수 있는 상태**입니다.
최종 경로는 여러 `k` 단계를 거치며 점진적으로 완성됩니다.

### 오해 3. `dist[i][k]`는 직접 간선이다

아닙니다.
`dist[i][k]`는 이미 이전 `k` 단계들에서 갱신된 **부분 최단거리**입니다.

### 오해 4. 반복문 순서는 아무거나 해도 된다

아닙니다.
플로이드-워셜은 **반드시 `k`를 바깥쪽 반복문**으로 두어야 합니다.

## 11. 시험/코테에서 기억할 한 줄 요약

### 핵심 정의

**`k`는 실제 경로 순서가 아니라, 중간 정점으로 사용할 수 있는 정점의 허용 범위다.**

### 핵심 해석

**플로이드-워셜은 경로를 따라가는 알고리즘이 아니라, 부분 최단거리들을 누적해서 전체 최단거리를 완성하는 DP다.**

### 질문에 대한 직접 답

`2 → 4 → 5 → 1 → 3` 같은 경로가 있더라도,
`k=1`에서 바로 `2 → 1 → 3`이 반영되지 않아도 문제없습니다.
나중에 `4`, `5` 등을 허용하는 단계에서 `2 → 1`이 먼저 갱신되고,
그 후 전체 `2 → 3` 최단거리도 최종적으로 반영됩니다.

---

## 12. 암기용 요약

```text
플로이드-워셜의 k는 실제 경로 순서가 아니다.
k는 "중간 정점으로 1~k번까지 허용"이라는 뜻이다.
따라서 어떤 최단 경로가 2→4→5→1→3처럼 생겨도,
처음부터 한 번에 반영되는 것이 아니라
부분 경로들이 먼저 갱신되고 나중에 전체 경로가 완성된다.
```

---

## 13. 함께 기억하면 좋은 포인트

* 플로이드-워셜은 **모든 정점 쌍 최단거리** 알고리즘이다.
* 시간복잡도는 `O(N^3)`이다.
* 음수 간선이 있어도 가능하지만, **음수 사이클이 있으면 최단거리 의미가 무너진다.**
* 경로 복원까지 하려면 `next` 배열 또는 `parent` 배열을 함께 관리할 수 있다.

---

## 14. 마무리

플로이드-워셜에서 가장 중요한 것은 **경로를 실제 순서대로 생각하지 않는 것**입니다.

이 알고리즘은

* "2에서 4로 가고"
* "그다음 5로 가고"
* "그다음 1로 가고"
* "그다음 3으로 간다"

처럼 실제 이동을 따라가는 방식이 아니라,

**중간에 사용할 수 있는 정점의 집합을 점점 확장하면서 전체 최단거리를 완성하는 DP**입니다.

이 관점만 잡히면,
왜 `k=1`에서 아직 안 되는 경로가 나중에 반영되는지 자연스럽게 이해할 수 있습니다.
