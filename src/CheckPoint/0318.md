## 0318

주사위 굴리기 문제
단순하게 직접 구현하는 문제이다.
정규표현식 만족하는지 확인하는 기능 .matches

dp_32175
피보나치 탑다운 방식
```
static long fib(int n) {
    if (n <= 1) return n;
    if (dp[n] != 0) return dp[n];  // 0이 아니면 이미 계산됨
    dp[n] = (fib(n - 1) + fib(n - 2)) % MOD;
    return dp[n];
}
```

근데 이 문제(32175)는 왜 -1이 필요하냐면, **dp값이 0이 되는 유효한 경우가 존재**하기 때문이에요. 
예를 들어 컵 크기가 [3]인데 높이가 1이면 `dp[1] = 0`이 정답인데, 
이걸 "아직 계산 안 함"과 구분할 방법이 없어요.
피보나치 → dp값이 항상 >= 1 (n >= 2)  → 0으로 미방문 구분 가능
이 문제  → dp값이 0일 수도 있음        → -1로 미방문 구분 필요 + 방문시 계산 전에 0으로 만들어 줘야 함
