# DP 상태 정의와 상태 전이 정리

## 1. DP란?

DP(동적 계획법)는 **큰 문제를 작은 문제로 나누고, 이미 계산한 결과를 저장해서 중복 계산을 줄이는 방법**이다.

핵심은 3가지다.

1. **상태 정의**: `dp[i]`, `dp[i][j]`가 무엇을 뜻하는가
2. **상태 전이**: 이전 상태들로 현재 상태를 어떻게 계산하는가
3. **초기값**: 가장 작은 문제의 답을 어떻게 둘 것인가

즉, DP는 단순히 점화식을 외우는 것이 아니라,
**무엇을 저장할지 정하고, 그 값이 어디서 오는지 설명할 수 있어야 한다.**

---

## 2. 상태란?

상태는 **DP 배열의 한 칸이 의미하는 상황**이다.

예를 들어:

* `dp[i] = i번째까지 보았을 때의 최적값`
* `dp[i][j] = i번째 물건까지 고려했을 때, 무게 j에서의 최대 가치`
* `memo[r][c] = (r,c)에서 도착점까지 가는 경로 수`

즉, `dp` 안에 저장되는 값 자체보다 더 중요한 것은
**그 인덱스가 어떤 의미를 가지는가**이다.

### 상태 정의를 잘해야 하는 이유

상태 정의가 애매하면:

* 점화식을 못 세운다
* 1차원으로 되는지 2차원이 필요한지 모른다
* 반복문 순서를 정할 수 없다
* top-down / bottom-up 중 어떤 방식이 자연스러운지 헷갈린다

그래서 DP에서는 항상 먼저 이렇게 말할 수 있어야 한다.

> `dp[i]`는 무엇이다.
> `dp[i][j]`는 무엇이다.

이게 출발점이다.

---

## 3. 상태 전이란?

상태 전이는
**현재 상태가 이전 상태들로부터 어떻게 계산되는지 정하는 규칙**이다.

쉽게 말하면:

> 현재 값을 만들기 위해, 어떤 이전 값들이 필요하고,
> 그것들을 어떤 방식으로 합칠 것인가?

이 질문에 대한 답이 상태 전이다.

예를 들어,

`dp[i] = dp[i-1] + dp[i-2]`

이 식은
현재 상태 `dp[i]`가 이전 상태 `dp[i-1]`, `dp[i-2]`로부터 만들어진다는 뜻이다.

즉,

* **상태 정의**: `dp[i]`가 무엇인지
* **상태 전이**: `dp[i]`를 어떻게 계산하는지

이 둘은 항상 같이 가야 한다.

---

## 4. 상태 전이를 왜 이해해야 하나?

많은 사람이 DP를 풀 때 점화식만 외우려고 한다.
하지만 진짜 중요한 건:

**현재 상태를 만들기 위해 어떤 정보가 필요한지 파악하는 것**이다.

이걸 이해하면:

* 점화식을 스스로 세울 수 있다
* 반복문 순서를 스스로 정할 수 있다
* 1차원 압축이 가능한지 판단할 수 있다
* DFS + 메모이제이션도 자연스럽게 이해된다

반대로 이걸 이해하지 못하면:

* 왜 `for (j--)`인지 모르고 외우게 된다
* 왜 어떤 문제는 2차원이 필요한지 모른다
* 왜 top-down은 되는데 bottom-up은 어렵게 느껴지는지 설명 못 한다

---

## 5. 예시 1: 피보나치

### 상태 정의

`dp[i] = i번째 피보나치 수`

### 상태 전이

`dp[i] = dp[i-1] + dp[i-2]`

왜냐하면 피보나치 수는 바로 앞의 두 수의 합이기 때문이다.

### 초기값

* `dp[0] = 0`
* `dp[1] = 1`

### 의미

현재 상태 `dp[i]`를 만들기 위해
이전 상태 `dp[i-1]`, `dp[i-2]`가 필요하다.

이게 가장 기본적인 상태 전이 예시다.

---

## 6. 예시 2: 계단 오르기

### 상태 정의

`dp[i] = i번째 계단까지 왔을 때 얻을 수 있는 최대 점수`

### 상태 전이

현재 `i`에 도달하는 방법을 생각하면,

* `i-2`에서 2칸 올라오기
* `i-3 -> i-1 -> i`로 오기

문제 조건을 반영하면 보통 다음처럼 된다.

`dp[i] = max(dp[i-2], dp[i-3] + stair[i-1]) + stair[i]`

### 초기값

문제 조건에 따라:

* `dp[1] = stair[1]`
* `dp[2] = stair[1] + stair[2]`

### 의미

현재 상태 `dp[i]`는
가능한 이전 이동 경로들의 최댓값으로부터 만들어진다.

즉, 상태 전이는 단순히 숫자를 더하는 것이 아니라
**문제의 이동 규칙을 식으로 옮기는 과정**이다.

---

## 7. 예시 3: 0/1 배낭

### 상태 정의

대표적인 정의 두 가지:

#### 2차원 정의

`dp[i][w] = i번째 물건까지 고려했을 때, 무게 w 이내에서 얻을 수 있는 최대 가치`

#### 1차원 압축 정의

`dp[w] = 현재까지 고려한 물건들로 무게 w 이내에서 얻을 수 있는 최대 가치`

### 상태 전이

현재 물건 `i`를 볼 때 선택지는 두 가지다.

* 넣지 않는다
* 넣는다

2차원에서는:

`dp[i][w] = max(dp[i-1][w], dp[i-1][w-weight[i]] + value[i])`

1차원 압축에서는:

`dp[w] = max(dp[w], dp[w-weight[i]] + value[i])`

### 왜 뒤에서 앞으로 갱신하나?

0/1 배낭에서는 **각 물건을 한 번만 사용 가능**하다.

만약 앞에서부터 갱신하면,
같은 물건으로 갱신된 `dp[w-weight[i]]`를 다시 참조해서
현재 물건을 여러 번 쓰는 효과가 생길 수 있다.

그래서 반드시:

`for (w = maxWeight; w >= weight[i]; w--)`

처럼 **뒤에서 앞으로** 가야 한다.

### 핵심

여기서 상태 전이를 이해했다는 것은
단순히 식만 아는 것이 아니다.

* 왜 이전 행을 참조해야 하는지
* 왜 1차원 압축이 가능한지
* 왜 반복문을 감소 방향으로 돌려야 하는지

이것까지 설명할 수 있어야 한다.

---

## 8. 예시 4: 완전 배낭

완전 배낭은 물건을 여러 번 사용할 수 있다.

### 상태 정의

`dp[w] = 무게 w 이내에서 얻을 수 있는 최대 가치`

### 상태 전이

`dp[w] = max(dp[w], dp[w-weight[i]] + value[i])`

식은 비슷하지만 반복문 방향이 다르다.

### 왜 앞에서 뒤로 갱신하나?

완전 배낭에서는 같은 물건을 여러 번 사용 가능하므로,
현재 물건으로 갱신된 값을 다시 참조해도 괜찮다.

그래서:

`for (w = weight[i]; w <= maxWeight; w++)`

처럼 **앞에서 뒤로** 간다.

### 0/1 배낭과의 차이

* 0/1 배낭: 한 번만 사용 가능 → 뒤에서 앞으로
* 완전 배낭: 여러 번 사용 가능 → 앞에서 뒤로

즉, 반복문 방향도 상태 전이의 일부다.

---

## 9. 예시 5: DFS + DP (내리막길)

### 상태 정의

`memo[r][c] = (r,c)에서 도착점까지 가는 경로 수`

### 상태 전이

현재 칸에서 더 낮은 칸으로만 이동할 수 있으므로,

`memo[r][c] = sum(memo[nr][nc])`

단, `(nr,nc)`는 현재 칸보다 낮은 인접 칸들이다.

### 기저 조건

도착점에 도달하면 경로 1개이므로:

`if (r == n-1 && c == m-1) return 1;`

### 의미

현재 상태 `(r,c)`는
다음 상태들 `(nr,nc)`의 값들을 합쳐서 만들어진다.

이 문제는 bottom-up으로 순서를 잡기 까다로울 수 있어서
DFS + 메모이제이션(top-down)이 자연스럽다.

즉,

* 아직 안 구한 상태면 재귀로 내려가고
* 한 번 구한 값은 memo에 저장해서 재사용한다

이 또한 상태 전이이다.

---

## 10. 1차원 DP가 되는지, 2차원이 필요한지

이건 DP에서 매우 중요하다.

### 1차원으로 가능한 경우

현재 상태를 계산할 때 필요한 정보가
인덱스 하나만으로 충분할 때.

예:

* `dp[i] = i번째까지의 최적값`
* `dp[w] = 무게 w에서의 최적값`

### 2차원이 필요한 경우

현재 상태를 계산하려면
하나의 인덱스만으로는 부족하고,
추가 정보가 더 필요할 때.

예:

`dp[i][j] = i번째 물건까지 고려했을 때, 무게 j에서의 최적값`

여기서는

* 몇 번째 물건까지 봤는지 `i`
* 현재 무게가 얼마인지 `j`

이 두 정보가 모두 필요하다.

### 판단 기준

현재 값을 계산할 때
무엇을 기준으로 구분해야 중복 계산이 사라지는가?

그 기준이 상태 차원이 된다.

---

## 11. top-down과 bottom-up

### top-down

* 큰 문제를 먼저 생각한다
* 필요한 작은 문제를 재귀로 호출한다
* 계산한 결과를 memo에 저장한다

예:

`dfs(r,c)`

장점:

* 문제 구조를 그대로 옮기기 쉬움
* 도달하는 상태만 계산함

단점:

* 재귀 호출 부담
* 순환 구조 주의

### bottom-up

* 작은 문제부터 차례대로 계산한다
* 반복문으로 테이블을 채운다

예:

`for (i=2; i<=n; i++) dp[i] = ...`

장점:

* 재귀 없이 안정적
* 계산 순서가 명확하면 빠름

단점:

* 순서를 잡기 어려운 문제도 있음

### 중요한 점

둘 다 본질은 같다.

둘 다 **상태와 전이**를 사용한다.
차이는 계산하는 순서일 뿐이다.

---

## 12. 상태 전이를 찾는 방법

문제를 보면 다음 순서로 생각하면 좋다.

### 1) 무엇을 저장할까?

* 최대값?
* 최소값?
* 경우의 수?
* 가능 여부?

예:

* 최대 점수
* 최소 비용
* 경로 수
* 도달 가능 여부

### 2) 그 값은 어디서 올까?

현재 상태를 만들기 위해
어떤 이전 상태들이 필요한지 생각한다.

예:

* 바로 이전 칸
* 이전 두 칸
* 더 작은 구간들
* 이전 물건까지의 결과

### 3) 초기값은 무엇일까?

가장 작은 문제의 답을 정한다.

예:

* `dp[0] = 0`
* 시작점은 1
* 도착점은 1

### 4) 계산 순서는 어떻게 할까?

전이가 성립하도록 반복문 방향이나 재귀 구조를 정한다.

예:

* 0/1 배낭은 뒤에서 앞으로
* 완전 배낭은 앞에서 뒤로
* DAG/내리막길은 DFS + memo

---

## 13. DP를 풀 때 스스로 던질 질문

문제를 풀 때 아래 질문을 계속 던지면 좋다.

1. `dp[i]` 또는 `dp[i][j]`는 정확히 무엇인가?
2. 현재 상태를 계산하려면 어떤 이전 상태가 필요한가?
3. 전이식은 무엇인가?
4. 초기값은 무엇인가?
5. 반복문은 왜 그 방향이어야 하는가?
6. 1차원으로 줄일 수 있는가?
7. top-down이 자연스러운가, bottom-up이 자연스러운가?

이 질문들에 답할 수 있으면
상태 전이를 제대로 이해한 것이다.

---

## 14. 자주 하는 실수

### 1) 상태 정의가 모호함

예:
`dp[i]`가 뭔지 정확히 설명하지 못함.

→ 점화식도 흔들린다.

### 2) 전이식만 외움

왜 그 식이 되는지 모르고 암기함.

→ 문제 조건이 조금만 바뀌어도 못 푼다.

### 3) 반복문 방향을 외움

왜 감소/증가해야 하는지 이해하지 않음.

→ 0/1 배낭, 완전 배낭에서 자주 틀린다.

### 4) 초기값을 대충 둠

기저 상태가 틀리면 전체가 무너진다.

### 5) 필요한 상태 차원을 빼먹음

1차원으로는 부족한데 억지로 줄이려다 실패한다.

---

## 15. 한 줄 정리

**상태 전이 이해 = 현재 DP 값을 만들기 위해 어떤 이전 상태들이 필요하고, 그 값들을 어떤 규칙으로 합쳐야 하는지 이해하는 것**

즉, DP는

* 무엇을 저장할지 정하고
* 그 값이 어디서 오는지 밝히고
* 그 순서대로 계산하는 과정

이다.

---

## 16. 암기용 요약

### DP 핵심 3요소

* 상태 정의
* 상태 전이
* 초기값

### 상태

DP 배열 한 칸이 의미하는 상황

### 상태 전이

현재 상태를 이전 상태들로부터 계산하는 규칙

### 기억할 질문

* `dp[i]`는 무엇인가?
* 어디서 오는가?
* 왜 그 식이 되는가?
* 왜 그 순서로 계산하는가?

### 배낭 반복문 방향

* 0/1 배낭: 뒤에서 앞으로
* 완전 배낭: 앞에서 뒤로

### DFS + DP

재귀로 필요한 상태를 내려가며 계산하고,
계산한 값은 memo에 저장해서 재사용

---

## 17. 마무리

DP를 잘한다는 것은
점화식을 많이 외운다는 뜻이 아니다.

진짜 중요한 것은:

* 문제를 보고 상태를 정의할 수 있는가
* 현재 상태가 어디서 오는지 설명할 수 있는가
* 그 전이가 맞도록 계산 순서를 잡을 수 있는가

이 세 가지다.

이 감각이 생기면,
새로운 DP 문제를 만나도 외운 풀이가 아니라
직접 상태와 전이를 설계해서 풀 수 있게 된다.
