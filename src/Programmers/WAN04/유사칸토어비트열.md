# 5진법 구간 분할 — 좌표변환 실수 정리

## 문제 요약

1~N번 숫자 중 **3이 포함된 숫자를 제외**하고, 구간 [l, r]에 속하는 숫자의 개수를 구한다.  
5진법 트리 구조로 분할하여 O(log N)에 풀려고 시도.

---

## 핵심 구조

n자리 5진법 트리에서, i번째 블록(i=1,2,4,5)이 차지하는 절대 구간:

```
블록 i의 절대 범위: (five[n-1] * (i-1), five[n-1] * i]
```

재귀 호출 시 이 블록 안의 상대 좌표로 변환해야 한다:

```
상대 l = 절대 l - five[n-1] * (i-1)
상대 r = 절대 r - five[n-1] * (i-1)
```

---

## 내가 했던 실수들

### 실수 1: 구간 완전 포함 케이스 누락

쿼리 구간 [l, r]이 블록 i를 **완전히 포함**하는 경우를 처리 안 했다.

```java
// 내 코드
if(l > five[n-1]*(i-1) && r <= five[n-1]*i) sum += all(n-1);  // l이 블록 안에 있을 때만
else if(l > five[n-1]*(i-1)) ...                               // l만 블록 안
else if(r <= five[n-1]*i) ...                                  // r만 블록 안
// → l이 블록 앞, r이 블록 뒤인 경우 → 3개 조건 전부 false → 블록 통째로 스킵!
```

**예시:** 블록 i=2의 범위가 (5, 10]인데 쿼리가 [4, 17]이면  
→ l=4 < 5, r=17 > 10 → 세 조건 모두 불만족 → 6~10 전체 누락

**올바른 처리:**

```java
long lo = five[n-1] * (i-1) + 1;
long hi = five[n-1] * i;
if (r < lo || l > hi) continue;           // 겹치지 않음 → 스킵

long nl = Math.max(l, lo);
long nr = Math.min(r, hi);

if (nl == lo && nr == hi) sum += all(n-1); // 완전 포함
else divide(nl - five[n-1]*(i-1), nr - five[n-1]*(i-1), n-1); // 부분 포함 → 상대좌표로
```

---

### 실수 2: 재귀 호출 시 절대좌표를 그대로 넘김

블록 i 안으로 재귀할 때 l, r을 **블록 기준 상대좌표로 변환**해야 하는데, 절대좌표를 그대로 넘겼다.

```java
// 내 코드 — 잘못된 부분
else if(l > five[n-1]*(i-1))
    divide(l % five[n-1] == 0 ? five[n-1] : l % five[n-1], five[n-1], n-1);
//   ↑ %로 얼추 계산하려 했지만 five[n-1] 경계에서 틀림

else if(r <= five[n-1]*i)
    divide(five[n-1], r, n-1);
//   ↑ l을 five[n-1]로 고정해버림 (완전히 틀린 값)
```

**예시:** 블록 i=4의 절대 범위가 (15, 20]이고 쿼리 l=4, r=17이면  
→ 블록 안에서 유효한 절대 범위는 [16, 17]  
→ 상대좌표: [16-15, 17-15] = [1, 2]  
→ divide(1, 2, n-1) 을 호출해야 함  
→ 내 코드는 divide(5, 17, 1)을 호출 → 전혀 다른 범위

**올바른 변환:**

```java
long base = five[n-1] * (i-1);
long nl = Math.max(l, base+1) - base;  // 상대 l
long nr = Math.min(r, base + five[n-1]) - base;  // 상대 r
divide(nl, nr, n-1);
```

---

### 실수 3: n==1 기저 조건 이중 계산

```java
// 내 코드
if(l<=3 && r>=3) sum += r-l;    // 3 포함 시 r-l 추가
sum += r-l+1;                    // 무조건 r-l+1 추가
// → 3이 포함되면 (r-l) + (r-l+1) = 전체 개수 * 2 - 1 이 되어버림
```

**올바른 처리:**

```java
sum += (r - l + 1) - (l <= 3 && r >= 3 ? 1 : 0);
```

---

## 올바른 divide 전체 구조

```java
static void divide(long l, long r, int n) {
    if (n == 1) {
        sum += (r - l + 1) - (l <= 3 && r >= 3 ? 1 : 0);
        return;
    }
    for (int i = 1; i < 6; i++) {
        if (i == 3) continue;
        long lo = five[n-1] * (i-1) + 1;   // 블록 i 절대 시작
        long hi = five[n-1] * i;             // 블록 i 절대 끝
        if (r < lo || l > hi) continue;      // 겹치지 않음

        long nl = Math.max(l, lo) - five[n-1]*(i-1);  // 상대 l
        long nr = Math.min(r, hi) - five[n-1]*(i-1);  // 상대 r

        if (nl == 1 && nr == five[n-1]) sum += all(n-1);  // 완전 포함
        else divide(nl, nr, n-1);
    }
}
```

## 교훈 요약

| 실수 | 원인 | 핵심 체크 |
|------|------|-----------|
| 완전 포함 케이스 누락 | if/else if 구조로 분기 → 세 조건 모두 false인 경우 존재 | 겹침 여부를 먼저 체크하고, 포함/부분으로 분기 |
| 절대좌표 그대로 재귀 | 블록 i가 five[n-1]*(i-1)에서 시작함을 잊음 | 재귀 전 반드시 `- base` 처리 |
| 기저 조건 이중 계산 | if와 이후 코드가 독립적으로 sum에 더함 | 하나의 식으로 합산 |
